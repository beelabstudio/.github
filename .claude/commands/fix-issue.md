# /project:fix-issue

Fix a GitHub issue or bug.

## Usage

/fix-issue <issue-number> or /fix-issue "<brief description>"

## Steps

1. Read the issue description or understand the problem
2. Explore the codebase to find relevant files
3. Identify the root cause
4. Implement the fix following code-style rules
5. Add or update tests
6. Run tests to verify the fix
7. Summarize the changes

## Output

- What was the issue?
- What was the root cause?
- What was the fix?
- Files modified
